# 命名空间
命名空间是云计算中用于资源隔离与权限管控的逻辑单元，核心作用是将集群内的资源（应用、存储、网络等）按用户、团队或业务场景分组，避免不同用户 / 业务的资源相互干扰，同时实现精细化的资源配额与权限管理。展示当前系统中的所有命名空间，对命名空间进行搜索、排序、分页浏览，并可对单条记录进行详情查看、编辑、删除等操作
## 1、基础信息
- 名称 / 所属可用区：填写命名空间名称（如 “我的个人空间 34”），选择所属可用区（如 “默认可用区”），明确资源的物理部署区域；
- 空间类型：选择 “私有”（仅创建者 / 授权用户可用）或 “公共”（平台用户共享），定义空间的访问范围；
- 说明：填写空间用途（如 “UserID:34 - 私有空间”），便于识别空间的业务归属。

## 2、资源配额配置
- CPU / 内存配额限制：设置命名空间内可使用的 CPU 核数（如 “12.00”）、内存容量（如 “24.00G”），避免资源过载；
- 存储配额：配置实验环境存储配额（如 “0.00”），限制空间内的存储资源使用量；
- 算力管理：查看 “剩余算力点”（如 “23.08078800”），监控空间内的算力消耗情况（若平台开启算力计量）。

## 3、安全与策略配置
- 安全账号 / 网络策略：开启 “是否使用安全账号”“是否使用网络策略”，增强命名空间的安全隔离与网络访问控制。

![alt text](image-7.png)

## 4、关联资源管理
- 数据卷快照：点击 “添加行”，配置命名空间内数据卷的快照（用于数据备份与恢复）；
- 部署 / 环境 / 服务关联：在 “部署 / 环境 / 服务” 标签页，添加该命名空间下的应用部署、运行环境、服务配置，实现资源的统一管理。
