# 存储快照类型
存储快照类型是用于定义存储快照的生成规则与管理策略的模板，核心作用是标准化存储数据的快照逻辑（如快照的存储后端、保留策略等），让用户 / 应用可以快速为存储资源（如 PV/PVC）创建符合规范的快照，实现数据的备份与恢复。
## 1、基础信息配置
名称：填写存储快照类型的标识名称（如示例中的csi-rbdplugin-snapclass），需与对应的存储后端插件（如 CSI-RBD）关联，清晰体现快照的适配场景。

## 2、编辑及删除快照类型
- 编辑快照类型：若存储后端插件升级或变更，可修改快照类型的关联配置，确保快照逻辑与存储系统兼容；
- 删除快照类型：若该快照类型不再使用，需先清理所有基于该类型的存储快照，再删除模板，避免快照管理失效。
